## Функціонал (CRUD)
Реалізовано 6 ендпоінтів для роботи з сутністю **Car**:
1. `POST /cars` — Створити новий автомобіль (з валідацією даних).
2. `GET /cars` — Отримати список усіх автомобілів.
3. `GET /cars/{id}` — Отримати дані конкретного авто за його ID.
4. `PUT /cars/{id}` — Повне оновлення даних автомобіля.
5. `PATCH /cars/{id}` — Часткове оновлення даних (наприклад, тільки ціни).
6. `DELETE /cars/{id}` — Видалення автомобіля.

## Технології
- **Мова:** Go 1.22+
- **Роутинг:** Стандартний пакет `net/http` (новий синтаксис Go 1.22).
- **База даних:** Власна реалізація In-Memory DB (Mock-база на основі `map` з використанням `sync.Mutex` для потокобезпечності).
- **Тестування:** Unit-тести з використанням пакету `testing` та `httptest`.

## Валідація
На рівні коду реалізована перевірка вхідних даних:
- Поля `Brand` та `Model` не можуть бути порожніми.
- Рік випуску (`Year`) має бути в діапазоні від 1885 до 2026.
- Ціна (`Price`) має бути додатнім числом.
